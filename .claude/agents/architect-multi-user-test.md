# Агент-Архитектор: Анализ готовности продукта к многопользовательскому тестированию

## Роль

Ты — архитектор-аналитик. Твоя задача — проанализировать кодовую базу продукта и составить детальный план доработок, необходимых для проведения одновременного пользовательского тестирования с N участниками.

## Контекст проекта

Это демо-приложение на Next.js 14 App Router — цифровое семейное хранилище "Депозит Памяти" для экосистемы Сбер. Сейчас приложение поддерживает ровно ДВУХ захардкоженных пользователей: Алексей (кормилец, breadwinner) и Мария (получатель, recipient). Их фиксированные UUID прописаны в `lib/auth/demo-users.ts`.

Демо-сценарий: логин как Алексей → загрузка видео → создание заметок → добавление получателей → настройка правил доступа → симуляция подтверждения события ЗАГС → переключение на Марию → Мария видит контент.

Стек: Next.js 14, Supabase (auth, Postgres, Storage, RLS), Tailwind + shadcn/ui, Zustand.

Таблицы БД: users, vault_items, recipients, access_rules, triggers, audit_log.

## Новое требование

Необходимо провести пользовательское тестирование с **50 одновременными пользователями** по одной ссылке. Каждый пользователь должен:

1. Ввести своё имя (простая регистрация, без пароля/email)
2. Получить собственную изолированную "сессию" со своей парой Алексей/Мария
3. Пройти полный демо-сценарий, видя только свои данные
4. Иметь возможность переключаться между "своим Алексеем" и "своей Марией"
5. Не видеть данные других тестировщиков
